= serveur web asynchrone
-faible utilisation de la mémoire
-maintenant aussi utilisé comme reverse proxy, cache HTTP, et load balancer.
-performant pour des sites gourmands en ressources
-installé dans /etc/nginx
-par défaut : /etc/nginx/sites-enabled
-modèle Event Driven : un seul thread qui réceptionne les requêtent et les dispatchent. Améliore les perf générales


*** création d'un nouveau fichier de config Nginx

1.créer un nouveau fichier dans /etc/nginx/sites-available
2.ajouter un lien symbolique dans /etc/nginx/sites-enabled grâce à la commande ln :
    ln -s lien-fichier-original lien-fichier-symbolique

*** modules

-core : Outils et directives essentiels
-event : configurer les mécanismes internes de la capacité de mise en réseau
-configuration : utiliser le système d’inclusion


*** fichiers de configuration

-nginx.conf : fichier de configuration globale du serveur. Les paramètres qui s'y trouvent doivent impacter l'ensemble du serveur ; on n'y touche normalement pas
-mime.types : liste des types MIME résolus par les extensions de fichiers
-sites-available : Contient les fichiers de configurations de vos sites ou services (un fichier par pré-occupation/site/service). Ce dossier est un dépôt ; les fichiers de configuration qui y sont ne sont pas pris en compte
-sites-enabled : doit contenir des liens symboliques vers les fichiers de site-available que l'on souhaite activer
-conf.d : dossier pour appliquer les paramètres communs à tous les sites. Peut également servir de la même façon que site-available (renommer ses fichiers ".conf" en ".disabled" pour désactiver les sites)

-proxy.conf
-Fastcgi.conf : FastCgi configuration


Par défaut, les fichiers de conf se trouvent dans :
    /etc/nginx/nginx.conf
    /usr/local/etc/nginx/nginx.conf
    /usr/local/nginx/conf/nginx.conf


contient : des directives et des contextes (équivaut à un scope), exemple :

worker_processes 2; # directive in global context

http {              # http context
  gzip on;          # directive in http context

  server {          # server context
    listen 80;      # directive in server context
  }
}
