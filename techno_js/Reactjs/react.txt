-2 modules React et ReactDOM
-2 classes : React.Component et React.PureComponent

>>> React
-React.Component : classe pour créer des composants React, n'implémente pas shouldComponentUpdate()
-React.PureComponent : créer des composants React, implémente shouldComponentUpdate()
-React.memo : comme PureComponent mais pour les composants créés à l’aide de fonctions 
-React.createElement()
-React.cloneElement()
-React.createFactory(type)
-React.isValidElement(object)
-React.Children.map()
-React.Fragment
-React.createRef
-React.forwardRef
-React.lazy
-React.Suspense


>>> ReactDOM : fournit des méthodes au DOM
-render() : Affiche un élément React au sein du nœud DOM
-hydrate() : Similaire à render(), mais sert à hydrater un conteneur dont le HTML a déjà été produit par ReactDOMServer
-unmountComponentAtNode() : Retire un composant React monté du DOM et nettoie ses gestionnaires d’événements et son état local
-findDOMNode() : renvoie l’élément DOM (natif au navigateur) qui a servi de point de montage.
-createPortal() : Crée un portail


>>> service worker
-script qui se lance entre la webpage et le réseau et qui sert de proxy. Il est utile en cas de perte du réseau



>>> syntaxe JSX
. expression JSX = contenu dynamique

// Avec JSX
<User first="John" last="Smith" />

// sans JSX
React.createElement(User, { first: 'John', last: 'Smith' })
