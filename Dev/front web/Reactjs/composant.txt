-éléments indépendants et réutilisables
-contient la balisage et la logique
-un composant peut s'écrire avec la syntaxe d'une fonction ou d'une classe
-comme des fonctions qui acceptent des props et renvoient des éléments react
-fonction composant = une fonction qui accepte des props et renvoit un élément react
-composition de composant : faire référence à d'autres composants en sortie
-composant pur fonctionnel = renvoit toujours la même chose
-separation of concerns
-props.children : passer des éléments enfants pour les comp qui ne connaissent pas leur enfant à l'avance (délégation de contenu)
-Les mises à jour de l’état peuvent être asynchrones
-Les mises à jour de l’état sont fusionnées
-affichage conditionnel = afficher un composant ou non en fonction de l'état de l'application
-en cas de changement, ReactDOM met à jour le strict nécessaire (les noeuds)


*** props
function Welcome(props) {
  return <h1>Bonjour, {props.name}</h1>;
}

class Welcome extends React.Component {
  render() {
    return <h1>Bonjour, {this.props.name}</h1>;
  }
}


*** constituant
-props = entrée
-state = état local, est privé
-méthodes de cycle de vie


*** règles
-tout composant React doit agir comme une fonction pure vis-à-vis de ses props
-les données descendent dans les enfants
-l’état remonte dans le parent : 
    -l’état est ajouté au composant qui en a besoin pour s’afficher
    -Si d’autres composants en ont également besoin, faire remonter l’état dans l’ancêtre commun le plus proche, puis descendre les données pour synchroniser les composants
-utiliser setState() pour modifier l'état
-Les composants utilisateurs doivent commencer par une majuscule
-nommer les props du point de vue du composant plutôt que de celui du contexte dans lequel il est utilisé


*** gestion des saisies de formulaire
-composant controlé : valeur saisie vérifiée
-composant non-controlé : valeur saisie non vérifiée
